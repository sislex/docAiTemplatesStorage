# Спецификация плейсхолдеров

В DOCX-шаблонах для подстановки динамических значений используются плейсхолдеры.

## Синтаксис

```
${name}
```

Где `name` соответствует регулярному выражению: `^[a-zA-Z_][a-zA-Z0-9_]*$`.

## Примеры корректных плейсхолдеров

```
${candidate_name}
${project}
${start_date}
${salary}
${_internal}
```

## Примеры некорректных плейсхолдеров

```
${123name}      — начинается с цифры
${my-field}     — содержит дефис
${}             — пустое имя
${name space}   — содержит пробел
```

## Правила использования

1. Каждый плейсхолдер шаблона должен быть объявлен в поле `placeholders` файла `metadata.json`.
2. При генерации документа все плейсхолдеры должны быть заменены на реальные значения.
3. Нераскрытые плейсхолдеры в итоговом документе считаются ошибкой.
4. Регистр `name` сохраняется: `${Name}` и `${name}` — разные плейсхолдеры.

## Обработка на backend

Backend принимает объект `Record<string, string>` и заменяет все вхождения `${key}` в тексте DOCX на соответствующие значения. Замена выполняется **только** для ключей из объекта; незнакомые плейсхолдеры оставляются без изменений.
