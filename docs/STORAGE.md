# Хранилище шаблонов

## Расположение

Все шаблоны хранятся в директории `storage/templates/`.

## Структура

Каждый шаблон занимает отдельную директорию, названную по его уникальному идентификатору:

```
storage/templates/
└── {id}/
    ├── template.docx   # Файл шаблона (DOCX)
    └── metadata.json   # Метаданные шаблона
```

## Правила

1. Имя директории (`{id}`) — единственный источник идентификатора; совпадает с полем `id` в `metadata.json`.
2. Оба файла (`template.docx` и `metadata.json`) **обязательны**. Директория без одного из них игнорируется.
3. Максимальный размер `template.docx` — **20 МБ** (задаётся через env `MAX_FILE_SIZE`).
4. БД не используется. Backend читает и пишет файлы напрямую.
5. Директория `storage/templates/` не индексируется при старте — данные читаются по запросу.

## Операции (backend REST API)

| Метод  | Путь                      | Описание                               |
| ------ | ------------------------- | -------------------------------------- |
| GET    | `/templates`              | Список всех шаблонов (все метаданные)  |
| GET    | `/templates/:id`          | Метаданные конкретного шаблона         |
| POST   | `/templates`              | Загрузка нового шаблона                |
| PUT    | `/templates/:id`          | Замена шаблона                         |
| DELETE | `/templates/:id`          | Удаление шаблона                       |
| POST   | `/templates/:id/generate` | Генерация DOCX с подстановкой значений |

## Git

Файл `storage/templates/.gitkeep` закоммичен для сохранения пустой директории.
Реальное содержимое (`storage/templates/*`) исключено из VCS через `.gitignore`.
